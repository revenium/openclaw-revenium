**## Setup section** must contain these elements in this order:

1. **Idempotency gate** (first thing): Check if `{baseDir}/config.json` exists.
   - If YES and user has NOT requested reconfiguration: setup is complete, proceed to Operation Guard.
   - If NO: run the Setup Flow below.
   - Use MUST/STOP mandatory language consistent with Phase 1 tone.

2. **Setup Flow** — a numbered procedure:
   - Step 1: Prompt the user for their Revenium API key. Use step-by-step collection (ask for key first, then budget details) to minimize friction — one thing at a time.
   - Step 2: Run `revenium config set key <API_KEY>`. If non-zero exit code: report error, tell user to run `/revenium` when ready, STOP. Do NOT write config.json.
   - Step 3: Prompt user for budget amount (numeric threshold, e.g., 5.00).
   - Step 4: Prompt user for budget period. Present the four options: DAILY, WEEKLY, MONTHLY, QUARTERLY.
   - Step 5: Auto-generate alert name from period: "OpenClaw {Period} Budget" where Period is title-cased (DAILY -> Daily, etc.). Do NOT ask the user for a name — this is a locked decision.
   - Step 6: Run `revenium alerts budget create --name "{name}" --threshold {amount} --period {PERIOD} --json`. If non-zero exit code: report error, tell user to run `/revenium` when ready, STOP. Do NOT write config.json.
   - Step 7: Extract the `"id"` field from the JSON response. This is a short alphanumeric string (e.g., "75BjG5"). Explicitly warn: do NOT use `anomalyId` from `budget get` responses — that is an integer and will cause HTTP 400.
   - Step 8: Write `{baseDir}/config.json` with `{"alertId": "<extracted-id>"}` — pretty-printed JSON. Use `python3 -c "import json; print(json.dumps({'alertId': 'ALERT_ID'}, indent=2))"` or equivalent. This MUST be the FINAL step — atomic setup, no partial state.
   - Step 9: Confirm to the user that setup is complete. Show the alert name, threshold, and period.

3. **Error handling pattern**: On any failure, report what went wrong, tell user to run `/revenium` when ready, and STOP. No retries. No partial config.json.

**## /revenium Command section** must contain:

1. **When setup is complete** (config.json exists):
   - Show current budget status: read alertId from config.json, run `revenium alerts budget get <alertId> --json`, display current spend vs threshold.
   - Offer to reconfigure: "Would you like to update your budget configuration?"

2. **When setup is NOT complete** (no config.json):
   - Run the Setup Flow from the Setup section.

3. **Reconfiguration Flow** (when user says yes to reconfigure):
   - Read existing alertId from `{baseDir}/config.json`
   - Run `revenium alerts budget delete {alertId} --yes` — if this fails (alert already deleted), log warning but continue
   - Delete `{baseDir}/config.json`
   - Run the full Setup Flow from scratch
   - This ensures no orphaned alerts in Revenium (locked decision)

- Keep instructions declarative — the agent is the runtime, not a shell script
- Use explicit variable names in instructions (e.g., "call this value ALERT_ID") so agents can track values through multi-step flows
- Reference `python3 -c` for JSON operations (more reliable than `jq` per research)
